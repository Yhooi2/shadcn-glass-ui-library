## Accessibility

- **ARIA**: `role="tooltip"`, `aria-describedby`
- **Keyboard**: ESC to dismiss
- **Focus**: Shows on focus for keyboard users

```tsx
<TooltipGlass content="Important information">
  <button aria-label="More information">
    <Info className="w-4 h-4" />
  </button>
</TooltipGlass>
```

## Best Practices

### ‚úÖ Do

- Keep tooltips concise (1-2 lines)
- Use for supplementary information
- Provide keyboard shortcuts in tooltips
- Explain disabled states
- Use for icon-only buttons

### ‚ùå Don't

- Don't use for critical information
- Don't make tooltips too long
- Don't use tooltips on mobile-only interfaces
- Don't hide important actions in tooltips
- Don't nest interactive elements in tooltips
