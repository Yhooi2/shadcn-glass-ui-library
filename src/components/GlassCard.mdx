| Intensity | Opacity | Use Case |

## Intensity Levels

### Subtle (8% opacity)

For backgrounds and non-intrusive overlays.

```tsx
<GlassCard intensity="subtle">
  <p>Subtle background card</p>
</GlassCard>
```

### Medium (15% opacity)

Default intensity for content cards.

```tsx
<GlassCard intensity="medium">
  <p>Standard content card</p>
</GlassCard>
```

### Strong (20% opacity)

For emphasized sections and important content.

```tsx
<GlassCard intensity="strong">
  <p>Emphasized card with high visibility</p>
</GlassCard>
```

## Best Practices

### ‚úÖ Do

- Use `intensity="medium"` as default for content
- Use `padding="default"` (24px) for most cases
- Choose glass variant based on content importance
- Combine with other glass components
- Use semantic HTML inside cards

### ‚ùå Don't

- Don't nest too many cards (max 2-3 levels)
- Don't use `intensity="strong"` for all cards
- Don't override opacity outside STANDARD_CARDS range
- Don't forget padding when adding content
- Don't make cards too small (min 200px width recommended)

## Accessibility

GlassCard is a presentational component. Ensure content inside is accessible:

```tsx
<GlassCard>
  <article aria-labelledby="card-title">
    <h3 id="card-title">Accessible Card</h3>
    <p>Content with proper semantic structure</p>
  </article>
</GlassCard>
```
